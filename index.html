Code Example: Form Validation

/* Style container when input is invalid and not empty */
                            .form-group:has(input:invalid:not(:placeholder-shown)) {
                            border: 2px solid #ef4444;
                            background-color: #fee2e2;
                            }

                            /* Style container when input is valid */
                            .form-group:has(input:valid:not(:placeholder-shown)) {
                            border: 2px solid #10b981;
                            background-color: #f0fdf4;
                            }

                            /* Style label based on input state */
                            .form-group:has(input:invalid) label {
                            color: #ef4444;
                            font-weight: 600;
                            }

                            /* Show error message only when invalid */
                            .form-group:has(input:invalid:not(:placeholder-shown)) .error-text {
                            display: block;
                            }

Why Container Queries Beat Media Queries

Media queries respond to viewport width:

/* Media query - viewport-based */
                            @media (min-width: 768px) {
                            .card {
                            flex-direction: row;
                            }
                            }

                            /* Problem: All cards change at same viewport width
                            regardless of their actual container size */

Container queries respond to container width:

/* Container query - container-based */
                            @container (min-width: 20rem) {
                            .card {
                            flex-direction: row;
                            }
                            }

                            /* Solution: Each card responds to its actual width
                            Same component works in sidebar, grid, or main content */

Benefits:

  • ‚úÖ Component responds to container, not viewport
  • ‚úÖ True component reusability across contexts
  • ‚úÖ Works with flexible grids and dynamic layouts
  • ‚úÖ No need for context-specific CSS classes
  • ‚úÖ Components are self-aware of available space
  • ‚úÖ Perfect for component-based frameworks (React, Vue)

Modern CSS Animations

Widely Supported

Animate display:none and height:auto using @starting-style, transition-behavior, and calc-size().

Demo 1: Animating display Property

Toggle visibility with smooth fade. Notice the element animates from display:none to display:block!

‚ú® This box smoothly fades in/out!

Previously impossible: animating display:none required JavaScript.

Demo 2: Smooth Accordion

Click to expand. Height animates smoothly from 0 to auto!

What is @starting-style?

@starting-style defines the initial state for entry animations. It works with display transitions to create smooth fade-in effects.

This enables animations that were previously impossible with pure CSS.

What is allow-discrete?

The transition-behavior: allow-discrete property enables transitions for discrete properties like display.

It makes the property wait until the end of the transition to change, making the animation visible.

Why is this useful?

Before these features, you couldn't animate elements appearing/disappearing with display:none.

You had to use visibility:hidden or opacity with workarounds. Now it's native CSS!

Code Example: Animating display

/* Element with display animation */
                            .box {
                            display: block;
                            opacity: 1;
                            transition:
                            opacity 0.4s ease-out,
                            display 0.4s allow-discrete;
                            }

                            /* Hidden state */
                            .box.hidden {
                            display: none;
                            opacity: 0;
                            }

                            /* Starting state for entry animation */
                            @starting-style {
                            .box {
                            opacity: 0;
                            }
                            }

Code Example: Smooth Accordion

/* Details/Summary accordion */
                            details {
                            transition: height 0.3s ease-out;
                            }

                            details[open] {
                            height: auto;
                            }

                            /* Smooth content reveal */
                            details[open] > .content {
                            animation: slideDown 0.3s ease-out;
                            }

                            @keyframes slideDown {
                            from {
                            opacity: 0;
                            transform: translateY(-10px);
                            }
                            to {
                            opacity: 1;
                            transform: translateY(0);
                            }
                            }

Why This Matters

The old problem:

You couldn't animate display: none to display: block because CSS transitions don't work with discrete properties. Elements would just pop in/out.

The old workaround (with JavaScript):

// Had to use setTimeout and remove display manually
                            function fadeOut(element) {
                            element.style.opacity = '0';
                            setTimeout(() => {
                            element.style.display = 'none';
                            }, 300); // Match transition duration
                            }

                            function fadeIn(element) {
                            element.style.display = 'block';
                            element.style.opacity = '0';
                            setTimeout(() => {
                            element.style.opacity = '1';
                            }, 10); // Force reflow
                            }

The new solution (pure CSS):

  • ‚úÖ @starting-style defines entry animation state
  • ‚úÖ allow-discrete enables display transitions
  • ‚úÖ No JavaScript timing hacks needed
  • ‚úÖ Smooth, declarative animations

scroll-state() Function

Emerging

The scroll-state() function enables CSS container queries based on scroll states like stuck, snapped, scrollable, and scrolled. Detect when sticky elements stick, when scroll containers are at certain positions, or track scroll direction‚Äîall without JavaScript.

Browser Support: Chrome 133+, Safari 18+ (requires container-type: scroll-state)

Your browser does not support scroll-state() queries. Please use Chrome 133+ or Safari 18+ to see the demos in action.

Demo 1: Sticky Header Detection

Scroll down to see the header change style when it becomes stuck to the top. The CSS detects the stuck state and applies different styling automatically.

Scroll to See Magic ‚ú®

This is a scrollable container with a sticky header. As you scroll down, watch how the header changes appearance when it sticks to the top.

What's Happening?

The container has container-type: scroll-state, which enables scroll-state queries. When the sticky header reaches the top and "sticks," the CSS query @container scroll-state(stuck: top) activates, applying new styles.

CSS-Only Detection

No JavaScript needed! The browser natively detects when the sticky element is stuck and allows you to style it accordingly. This is perfect for navigation bars, table headers, and any sticky UI elements.

Performance Benefits

Since this is handled by CSS, it's highly performant. No scroll event listeners, no layout thrashing, no JavaScript performance overhead. The browser handles everything efficiently.

Direction-Aware Header

The Power of CSS Scroll Detection

Modern CSS can now detect scroll direction without any JavaScript. As you scroll down this article, watch the header disappear and a "Back to Top" button appear.

The scroll-state(scrollable) query detects whether content can be scrolled in a given direction, with values like top, bottom, left, and right. When scrollable: top is true, it means you can scroll up‚Äîindicating you've already scrolled down!

Why This Matters

Traditionally, detecting scroll position required JavaScript event listeners, scroll calculations, and state management. Now, CSS handles it natively with better performance and simpler code.

This technique is perfect for:

  • Auto-hiding navigation bars
  • Showing/hiding floating action buttons
  • Context-aware UI elements
  • Reading mode optimizations

Browser Support

The scrollable state is available in Chrome 133+ and Safari 18+. Other states include stuck for sticky positioning and snapped for scroll-snap detection.

Code Example: Sticky Header Detection

/* The sticky element itself is the container */
                            .sticky-header {
                            container-name: sticky-heading;
                            container-type: scroll-state;
                            position: sticky;
                            top: 0;

                            /* Style the content inside */
                            .header-content {
                            padding: 1rem 1.5rem;
                            background: #f9fafb;

                            @container sticky-heading scroll-state(stuck: top) {

// Detect sticky state with JavaScript

                            // Track scroll direction
